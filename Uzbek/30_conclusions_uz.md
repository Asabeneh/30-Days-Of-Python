# 📘 30-kun

## 30 kunni yakunlaymiz

🎉 **TABRIKLAYMIZ!** 🎉

Siz 30 kunlik Python challenge'ini muvaffaqiyatli yakunladingiz! Bu oddiy yutuq emas — har kuni yangi mavzuni o'rganib, kod yozib, mashqlarni bajarib kelgan siz — haqiqiy dasturchisiz.

> Agar barcha mashqlar va loyihalarni bajarsangiz — siz endi **ma'lumotlar tahlili (data analysis), sun'iy intellekt (machine learning)** yoki **veb-dasturlash (web development)** yo'nalishlariga o'tishga tayyor bo'ldingiz.

## Nima o'rgandingiz

30 kun davomida siz quyidagi mavzularni o'rgandingiz:

| Kun | Mavzu | JS bilan taqqoslash |
|-----|-------|---------------------|
| 1 | Python'ga kirish, o'rnatish | `console.log` ↔ `print()` |
| 2 | O'zgaruvchilar, ichki funksiyalar | `let/const` ↔ e'lon qilmasdan; `parseInt` ↔ `int()` |
| 3 | Operatorlar | `&&/||/!` ↔ `and/or/not`; `True/False` katta harf |
| 4 | Satrlar | Template literals ↔ f-satrlar |
| 5 | Ro'yxatlar (List) | Array'ga o'xshaydi, `append/pop/sort` bir xil |
| 6 | Kortejlar (Tuple) | Muzlatilgan ro'yxat — JS'da to'g'ri ekvivalenti yo'q |
| 7 | To'plamlar (Set) | JS Set'dan kuchliroq: `union/intersection/difference` |
| 8 | Lug'atlar (Dictionary) | JS Object ↔ `keys/values/items()` |
| 9 | Shartlar | `else if` ↔ `elif`; ternary teskari tartib |
| 10 | Sikllar | `for...of` ↔ `for...in`; `range()` ↔ klassik `for` |
| 11 | Funksiyalar | `function` ↔ `def`; `this` ↔ `self` (majburiy!) |
| 12 | Modullar | ES modullar ↔ `import/from...import` |
| 13 | List comprehension | `map().filter()` ↔ `[x for x in list if cond]` |
| 14 | Yuqori tartibli funksiyalar | Callbacks, closure, dekoratorlar |
| 15 | Xatolar turlari | `try/catch` ↔ `try/except`; JS'da silent, Python'da aniq |
| 16 | Sana va vaqt | `new Date()` ↔ `datetime.now()`; oy 1 dan boshlanadi! |
| 17 | Istisnolar | `try/except/else/finally`; `enumerate`, `zip` |
| 18 | Muntazam ifodalar | `/pattern/` ↔ `r'pattern'`; deyarli bir xil sintaksis |
| 19 | Fayl bilan ishlash | `open()`; `JSON.parse/stringify` ↔ `json.loads/dumps` |
| 20 | Paket menejeri | `npm` ↔ `pip`; `package.json` ↔ `requirements.txt` |
| 21 | Klasslar va obyektlar | `class/extends/super` bir xil; `new` yo'q; `self` majburiy |
| 22 | Veb-skreyping | `querySelector` ↔ `find()`; BeautifulSoup |
| 23 | Virtual muhit | `node_modules` ↔ `venv`; lekin qo'lda yoqish kerak |
| 24 | Statistika va NumPy | Vektorizatsiya: `arr + 10` — `.map()` shart emas |
| 25 | Pandas | Jadval ma'lumotlari; `[{...}, {...}]` ↔ DataFrame |
| 26 | Flask — Veb | `app.get()` ↔ `@app.route()`; Jinja2 ↔ EJS |
| 27 | MongoDB | JSON hujjatlar; `find/insert/update/delete` |
| 28 | API va HTTP | `fetch()` ↔ `requests.get()`; GET/POST/PUT/DELETE |
| 29 | RESTful API yaratish | To'liq CRUD API Flask + MongoDB bilan |

## Keyingi qadamlar

Siz endi uch yo'nalishdan birini tanlashingiz mumkin:

### 1. Ma'lumotlar tahlili va Sun'iy intellekt
- **NumPy** va **Pandas** — 24-25-kunda boshladingiz
- **Matplotlib / Seaborn** — grafiklar chizish
- **Scikit-learn** — machine learning
- **Jupyter Notebook** — interaktiv muhit

### 2. Veb-dasturlash (Backend)
- **Flask** — 26-29-kunda boshladingiz
- **Django** — yirik loyihalar uchun, kuchliroq framework
- **FastAPI** — zamonaviy, tez, avtomatik API hujjatlash
- **PostgreSQL / MySQL** — SQL bazalari

### 3. Avtomatizatsiya va Skriptlar
- **Selenium** — brauzer avtomatizatsiyasi
- **Scrapy** — industrial darajadagi veb-skreyping
- **Celery** — fon vazifalari
